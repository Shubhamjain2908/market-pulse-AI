## Strict Guardrails

| Guardrail | Rule | Enforced In |
|---|---|---|
| **Deep loss full review** | Unrealised loss > 20% → mandatory full LLM review, never lite path | `evaluate-trades.ts` threshold check |
| **RSI overbought ADD block** | RSI_14 > 70 OR price within 3% of 52W high → block ADD, output HOLD | Portfolio analyser system prompt Rule 6 + code guard |
| **Low volume ADD block** | `volume_ratio` < 0.5 → block ADD | Portfolio analyser system prompt Rule 7 |
| **No duplicate ADD** | Symbol has ≥1 open paper trade → block ADD, output HOLD with note | `portfolio-analyser.ts` pre-check (newly added) |
| **ADD pullback requirement** | ADD requires ≥1 ATR pullback from prior entry OR confirmed breakout on vol > 1.5× | Portfolio analyser system prompt Rule 9 (newly added) |
| **Averaging-down disclosure** | If position at loss: state (a) % gain to breakeven, (b) whether stop allows recovery room | Portfolio analyser system prompt Rule 8 |
| **No macro hallucination** | No FII/DII/USD/crude in stock-specific thesis unless directly tied to that stock's economics | All agent prompts |
| **No financial hallucination** | Never invoke data not present in provided context | All agent prompts |
| **Confidence range** | Full 1–10 scale. Strong tech + fundamentals = 7–8. Pure tech, weak fundamentals = 3–4. False momentum flag = max 5. Catalyst-event theses are hard-capped in code at max 6. | Thesis generator system prompt + post-LLM clamp |
| **ETF/SGB RSI exclusion** | LIQUIDCASE, GOLDBEES, GOLDCASE, SILVERBEES, NIFTYBEES, JUNIORBEES, SGBs — skip RSI/volume signals entirely | `config/etf-exclusions.json` + portfolio analyser (newly added) |
| **Regime gate absolute** | momentum_mf: no entries if regime ≠ BULL_TRENDING. No exception. | `momentum-rebalance.ts` pre-check |
| **Regime strategy gate fail-closed** | `isStrategyAllowed(strategy_id, regime)`: missing `(strategy_id, regime)` row in `regime_strategy_gate` → **DISALLOWED** (`log.warn`, return false). Seed `config/strategy-gates.json` must cover every tuple including CRISIS rows. | `src/db/regime-queries.ts` |
| **mom_false_flag entry block** | `momentum_mf` rebalance: `mom_false_flag = 1` → hard-block paper trade insert (`falseFlagBlocked` counter). Thesis confidence cap ≤ 5 still applies when thesis path runs elsewhere. | `src/strategies/momentum-rebalance.ts` |
| **Portfolio rank-decay routing** | Portfolio analyser only applies `mom_rank` action overrides when entry origin is `momentum_mf` (paper ledger or inferred). First breach (`mom_rank > exit_rank_threshold`) becomes `TRIM` with `GUARDRAIL_OVERRIDE[...]`; severe breach (`threshold + 5`) becomes `EXIT`. Non-momentum origins may see rank context but do not exit on rank alone. | `src/agents/portfolio-strategy-guardrails.ts` |
| **Portfolio entry-source inference** | Holdings without paper trades infer origin from recent screens (365d), open momentum sleeve, or thesis `trigger_reason` text; fallback `unknown`. Feeds strategy guardrails + LLM position context. | `src/agents/portfolio-strategy-guardrails.ts` |
| **Quality-GARP portfolio deterioration** | `quality_garp` holdings: 2+ inverse gate flags → `TRIM`; promoter selling + profit decline or 4+ flags → `EXIT`. Prefix `GUARDRAIL_OVERRIDE[strategy=quality_garp,...]`. | `src/agents/portfolio-strategy-guardrails.ts` |
| **Catalyst hold-window exit** | `catalyst_entry` holdings: open paper row past `max_hold_days` → `TRIM`; inferred/screen-only uses `expected_earnings_date + 2` calendar days (fallback: `screen.date + days_to_earnings + 2`) vs analysis date → `TRIM`; severe overrun (5+ days past window end or P&L < -5%) → `EXIT`. | `src/agents/portfolio-strategy-guardrails.ts` |
| **mom_false_flag net profit check** | `momentum_mf` false-flag fires when `z_12_1 > 0.674` AND (`profit_growth_yoy < -5%` OR `net_profit_ttm < 0`). The second condition blocks loss-making companies whose YoY looks positive due to loss-narrowing base effect (e.g. IDEA June 2026: YoY +13% but absolute profit still negative). Requires `net_profit_ttm` column (migration 0023) populated via Yahoo snapshot (`financialData.netIncomeToCommon` when present; else `fundamentalsTimeSeries` trailing TTM, preferring `normalizedIncome`). NULL `net_profit_ttm` → condition treated as false (fail-open on missing data). | `src/rankers/momentum-ranker.ts`, `src/ingestors/yahoo-snapshot-ingestor.ts` |
| **Earnings calendar retain on empty** | Sunday Yahoo earnings sync: `replaceMomentumEarningsCalendarForSymbol(db, symbol, [])` retains existing `earnings_calendar` rows (no DELETE on empty Yahoo response). | `src/db/momentum-queries.ts`, `src/ingestors/yahoo/earnings-ingestor.ts` |
| **alreadyOwned filter** | Skip symbol in AI Picks if currently held in Kite portfolio **or** symbol has any OPEN `paper_trades` row (any signal type) | Thesis generator input preprocessing |
| **Corporate action nominal adjust** | Daily automated stage runs after ingest and before enrich; for OPEN `paper_trades` pulls Yahoo `splitHistory` over last 5 IST days and applies nominal divide exactly once via `INSERT OR IGNORE` + `run().changes` | `src/ingestors/corporate-actions.ts` |
| **Gap-down circuit breaker** | If session `open < (prior NSE EOD close × 0.70)`, skip stop-out + target checks for **that bar only**; still run persistence/time-stop and log with recent corporate-action context (`hasCorporateActionInRange`) | `src/scripts/evaluate-trades.ts`, `src/db/queries.ts` |
| **Gap-up circuit breaker** | If session `open > (prior NSE EOD close × 1.30)`, suppress `highest_close_since_entry` update for **that bar only**; stop-out + target checks still run; persist real watermark not gap-inflated close | `src/scripts/evaluate-trades.ts`, `src/db/queries.ts` (`getPrevClose`) |
| **Fixed stop path** | `paper_trades.stop_type='fixed'` bypasses ATR trailing math/log writes; evaluator uses static `stop_loss`/`target` under the same circuit-breaker envelope | `src/scripts/evaluate-trades.ts` |
| **Stale Kite holdings** | Kite-backed holdings whose snapshot `as_of` is before the expected NSE session (`lastOpenOnOrBefore(run date)`) → skip all portfolio LLM/lite; persist `HOLD` placeholders with `STALE_HOLDINGS` trigger; warn + briefing banner. Manual-only book skips this. | `src/agents/portfolio-analyser.ts`, `src/briefing/composer.ts`, `src/briefing/template.ts` |
| **Partial pipeline briefing** | If the **latest** `pipeline_runs` status for `enrich`, `regime`, or `screen` on `run_date` is `failed` (`getPipelineHealth` — not any historical failure row), briefing renders degraded: failure banner + mood/FII/DII + regime card only when regime's latest status succeeded. **Omit** screens, AI picks, portfolio, momentum. Thesis/portfolio-analysis LLM optional stages do not trigger degraded mode. | `src/db/pipeline-queries.ts`, `src/briefing/composer.ts`, `src/briefing/template.ts` |
| **Briefing skip-ai + persisted theses** | `brief --skip-ai` skips mood LLM; AI Picks still render when `theses` rows exist for the session (`resolveAiPicksStatus` prefers thesis count over skip flag). | `src/briefing/composer.ts`, `src/briefing/template.ts` |
| **Fundamentals percent normalization** | `roe` / `roce` / `dividend_yield` from Yahoo decimals (`0.18`) scaled to percent at screen read when `|v| < 1`; Screener percent rows unchanged. | `src/analysers/signal-provider.ts` |
| **Signals 90-day read window** | `signals` technical reads (screener DB provider + `getLatestSignals*`) require `date >= date(as_of, '-90 days')` on outer and inner query parts; zero rows → empty / null — no fallback to older rows | `src/analysers/signal-provider.ts`, `src/agents/portfolio-trigger.ts` |
| **Quality-GARP hard null block** | `quality_garp` v2 must hard-block when any required gate input is missing (`pe`, `pb`, `third_roe`, `latest_roce`, `debt_to_equity`, `peg`, `sma_50`, `close`); only `promoter_holding_change_qoq` and the OPM stability input are fail-open on NULL | `src/analysers/stock-screener.ts` |
| **Quality-GARP OPM stability** | Gate 11: trailing 4-quarter `quarterly_fundamentals.opm_pct` population std-dev > 5% → hard block (`opm_stability` funnel fail). <4 quarters of data → fail-open and increment `opm_skipped` (gate skipped only when symbol would have passed all other gates). | `src/analysers/quality-garp-screen.ts`, `getTrailingOpmStdDev` in `src/db/queries.ts` |
| **Quality-GARP pledge gate** | Gate 12: `promoter_pledge.pct_shares_pledged` > 15% → hard block when `QUALITY_GARP_PLEDGE_GATE=1`. Default `'0'`: shadow-only (`pledge_shadow` funnel counter); no pledge row → fail-open (`pledge_skipped`). Portfolio pledge flags gated behind same env — no TRIM/EXIT escalation until gate on. | `src/ingestors/nse/pledge-fetcher.ts`, `quality-garp-screen.ts`, `portfolio-strategy-guardrails.ts`, `QUALITY_GARP_PLEDGE_GATE` in `env.ts` |
| **AI_PICK rubric gate (shadow)** | `AI_PICK_RUBRIC_GATE=0` (default): rubric is computed and persisted for every thesis, but `confidence ≥ 6` remains the sole admission check. `facts` include `rubricTotal`, `rubricEligible`, `rubricDisagreesWithConfidence` for calibration. `AI_PICK_RUBRIC_GATE=1`: replaces `confidence < 6` early-return with `rubricTotal < AI_PICK_RUBRIC_MIN`. | `src/briefing/ai-pick-gate.ts`, `src/config/env.ts` (`AI_PICK_RUBRIC_GATE`, `AI_PICK_RUBRIC_MIN`) |
| **Concall intel is advisory-only** | Concall intelligence (sentiment, guidance, credibility) feeds thesis context (`buildStockContext`) and briefing cards but must **never** feed `portfolio-strategy-guardrails.ts` flags, screen gates, or strategy outcomes until the next clean BULL_TRENDING / GTT re-baseline. The `CONCALL_ANALYSIS_ENABLED` env flag defaults to `'1'`. | `src/agents/concall-analyser.ts`, `src/agents/thesis-generator.ts` (`buildStockContext`), `src/briefing/composer.ts` |
| **Quality-GARP PIT replay** | Live screen uses exact snapshot `as_of = date` (default). `cli backtest` for `quality_garp` passes `pointInTimeFundamentals: true` → `getQualityGarpFundamentals` annual/snapshot `as_of <= date`. Does not mutate `fundamentals`. | `src/db/queries.ts`, `src/backtest/harness.ts` |
| **Quality-GARP QDS gate** | Gate 13: 6-signal Quality Decay Score from `quarterly_fundamentals`. Hard block at ≤ 3 (P10-derived); soft warning (passes with `qds_warning: true`) at score = 4; fail-open when <5 quarters; bypassed entirely in CRISIS regime. Funnel counters: `qds`/`qds_warning`/`qds_skipped`. | `src/analysers/quality-garp-screen.ts`, `src/db/queries.ts` (`getQualityDecayScore`) |
| **Quality-GARP regime-aware thresholds** | Gates 4/9/10/11 (PE, PEG, RSI, SMA50) resolved per regime via `resolveGarpThresholds(regime?)` in `quality-garp.ts`. CHOPPY (default) returns existing constants — **zero behaviour change**. BULL: RSI 55, SMA50 8%, PEG 1.4, PE 40. BEAR: RSI 40, SMA50 3%, PEG 1.0, PE 28. CRISIS: RSI 35, SMA50 0%, PEG 0.9, PE 22. **Do NOT vary** ROE, ROCE, D/E, PB, or OPM thresholds by regime — these are fundamental quality floors. Active thresholds persisted in `matched_criteria.regime_thresholds` on every pass. BULL/BEAR/CRISIS values are calibration-pending placeholders until 6-month screen-history regime audit runs. | `src/analysers/quality-garp.ts`, `src/analysers/quality-garp-screen.ts` |
| **Quality-GARP universe** | Live + backtest screen `yahoo_annual` DISTINCT (~241), not watchlist. Funnel JSONL includes `universe_scope`. Tests/backtests may pass explicit `symbols` → `override`. | `src/analysers/quality-garp-universe.ts` |
| **Option A backtest regime gate** | Default **proxy**: quotes-only coarse regime (`regime-proxy.ts`), gate = ≥252 `NIFTY_50` rows before `--from`. **`--regime-source daily`**: ≥80% of benchmark trading days must have `regime_daily` (see `backtest-queries`). Momentum factors use `adj_close`-aligned series. Phase 1 initial-ATR sweep: `--sweep-initial-stop` / `--initial-multiplier` on `momentum-mf`. | `src/backtest/runner.ts`, `src/backtest/regime-proxy.ts` |
| **GTT activation** | No live GTT / order routing until all gates in `docs/gtt-activation-criteria.md` pass (post-fix tranche `source_date >= 2026-05-14`, per-signal expectancy floors, `BULL_TRENDING` at activation). Healthcheck logs tranche metrics daily; empty tranche does not fail the run. | `docs/gtt-activation-criteria.md`, `deploy/healthcheck.ts` |
| **momentum_mf initial stop** | Entry stop uses `position_sizing.atr_multiplier` from `momentum-config.json` (**2.5×**). Hard −8% floor (`entry × 0.92`) still binds. Re-enrich + `scripts/audit-atr-alignment.mts` before deploy after quote backfill. | `config/momentum-config.json`, `momentum-rebalance.ts` |
| **AI_PICK stop distance** | After gate pass: `minDist = max(entry×2%, atr14×1)`; block when `minDist > entry×8%`; widen thesis stops closer than `minDist` (`ai_pick_stop_normalized` warn); 8% floor remains final backstop with explicit `ai_pick_stop_floor_applied` audit logging | `src/briefing/ai-pick-stop.ts`, `paper-trade-writer.ts` |
| **AI_PICK admission gate** | `AI_PICK` paper-trade insert requires `evaluateAiPickEligibility()` pass: `confidence ≥ 6`, no fresh `mom_false_flag = 1`, and one of Path A (same-day non-`golden_cross` screen), Path B (alert-backed breakout), or Path C (`golden_cross` tiers in `CHOPPY` with fresh rank/false-flag ≤2 sessions via `NIFTY_50` calendar). Stale false flags are logged but do not overblock Path A/B; rank-dependent paths fail closed. Outside `CHOPPY`, `golden_cross`-only needs Path A/B. Thesis cards still render in briefing. | `src/briefing/ai-pick-gate.ts`, `paper-trade-writer.ts` |
| **Cross-strategy entry dedup** | A symbol with any OPEN `paper_trade` blocks new entries from any other `signal_type`. Closed trades do not block. | `src/db/queries.ts` (`hasOpenPaperTradeForSymbol`) |

Book = latest `portfolio_holdings` mark-to-market sum (`resolveBookValueInr`). Persisted as `paper_trades.position_weight_pct`. GTT gate still reads unweighted expectancy; healthcheck logs weighted shadow. | `src/strategies/position-sizer.ts`, `paper-trade-writer.ts`, `momentum-rebalance.ts` |

Momentum `max_per_sector` (3) remains a live per-sleeve block. | `position-sizer.ts`, `config/momentum-config.json` |
